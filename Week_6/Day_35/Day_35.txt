# 1. Impurity-Based Feature Importance

# Tree models naturally generate importance because:

# splitting reduces impurity

# Important features:

# reduce uncertainty
# create purer nodes
# improve prediction
# Impurity Intuition

# High impurity:

# mixed/confused node

# Low impurity:

# cleaner separation
# Importance Measures
# Type	Meaning
# Weight	split frequency
# Gain	impurity reduction contribution
# Cover	no. of samples affected
# Most Useful
# Gain importance

# because it reflects:

# actual contribution quality

# not just frequency.

# 2. Importance ≠ Causality

# Feature importance means:

# predictive usefulness inside model

# NOT:

# real-world causal effect

# Very important distinction.

# 3. Importance Is Relative

# Features compete during splitting.

# Importance depends on:

# correlations
# interactions
# redundancy
# tree structure

# Thus:

# rankings are relative

# not absolute truth.

# 4. Correlated Features Problem

# Correlated features:

# contain overlapping information

# Trees may:

# choose one
# ignore/mask others

# Result:

# unstable rankings
# misleading importance
# masking effect
# Masking Effect

# One correlated feature hides importance of another.

# Thus:

# low importance ≠ uselessness
# 5. Global vs Local Importance
# Global Importance

# Answers:

# “What matters overall?”

# Dataset-wide behavior.

# Local Importance

# Answers:

# “Why THIS prediction happened?”

# Prediction-specific reasoning.

# Important Insight

# Globally weak features can become:

# locally critical

# in:

# rare subgroups
# edge cases
# special contexts
# 6. SHAP Intuition

# SHAP =

# prediction contribution explanation system

# Inspired from:

# Shapley values (game theory)
# SHAP Core Formula
# Prediction = Baseline + Feature Contributions
# Baseline

# Baseline =

# average/default prediction

# SHAP explains:

# how features moved prediction away from baseline
# SHAP Contributions
# SHAP Value	Meaning
# Positive	pushes prediction upward
# Negative	lowers prediction
# SHAP Advantages

# ✅ local explanations
# ✅ directional reasoning
# ✅ contextual interpretation
# ✅ prediction-level analysis

# Important SHAP Insight

# Same feature can:

# increase prediction in one case
# decrease in another

# because SHAP is:

# contextual/local
# 7. Fraud Interpretation

# Fraud predictions arise through:

# different behavioral pathways

# Examples:

# huge amount fraud
# account takeover
# impossible travel
# midnight anomaly

# Thus:

# same fraud score can have different explanations
# 8. Leakage Detection

# Features like:

# Chargeback_Status

# may indicate:

# future data leakage

# Leakage causes:

# fake confidence
# inflated train performance
# poor real-world generalization
# 9. Explainability Helps Debugging

# Interpretation helps detect:

# leakage
# memorization
# spurious correlations
# subgroup failure
# unstable optimization

# Predictive vs Causal
# Predictive	Causal

# Global vs Local
# Global	Local
# overall behavior	specific prediction

# Raw Importance vs SHAP
# Raw Importance	SHAP
# overall usefulness	contribution to prediction
